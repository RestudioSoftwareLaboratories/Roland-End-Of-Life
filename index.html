Official Statement of Permanent Discontinuation — Effective June 19, 2026

Restudio Software Laboratories hereby formally announces the permanent discontinuation of the Roland Script programming language, including the termination of all development, technical support, and maintenance activities. The project, together with its associated editor, is hereby declared permanently discontinued and removed from all future development plans of the laboratories, effective June 19, 2026.

This decision is final, strategic, and irreversible. In the interest of transparency, Restudio Software Laboratories is publishing the technical, architectural, and practical reasons that led to the permanent retirement of the project and the withdrawal of its associated tools.

I. Technical and Architectural Reasons

Cancellation of Core Features Planned for Version 3.0

During the planning phase of Roland Script v3.0, it became evident that the architecture of the existing interpreter was fundamentally incapable of supporting several essential language features. As a result, these features were permanently cancelled before implementation, including:

  • Protected constants (prove var)
  • Logical comparison and relational operators (younger than, greater than)
  • Advanced logical operators (and, or, xor, not)
  • Reusable programming functions (act! =, repass!)
  • Alternative conditional branches (other:, wrong other:) and multi-selection structures (SLbell)
  • Range-based iteration (<range>)

The limitations of both the interpreter and Restudio Script Editor 1.0 (RSE 1.0) prevented the language from evolving beyond a highly experimental state, making it unsuitable for practical software development.

Absence of Standard Primitive Data Types

Roland Script relies on a non-standard and simplistic parsing model. For example, textual values (Words) and characters are assigned without quotation marks (e.g., X = ammar). This parsing approach creates ambiguity between identifiers and literal values, preventing the implementation of advanced language features such as robust type handling, symbol resolution, and sophisticated function systems.

Pointerless Array Implementation

The language's array mechanism (line for class) is based on manually defined static variables and lacks support for dynamic indexing or pointer-based access. Consequently, arrays function primarily as containers for displaying stored values rather than as fully manipulable data structures.

Conditional Execution Limitations

The interpreter does not implement an Abstract Syntax Tree (AST). Instead, it processes source code sequentially on a line-by-line basis and skips lines using currentLineIndex++ whenever a conditional expression evaluates to false. This architecture assumes that every conditional block occupies only a single line, making nested or complex conditional structures impractical.

II. Editor Status and Source Code Closure

Retirement of Restudio Script Editor 1.0

The official editor for Roland Script was Restudio Script Editor 1.0 (RSE). The editor lacked essential development capabilities such as IntelliSense, structural code analysis, and integrated debugging tools. Due to these technical limitations, RSE 1.0 has been permanently retired.

⛔ Source Code Closure: The source code of both Roland Script and Restudio Script Editor 1.0 has been permanently closed and is no longer available for public distribution.

Restudio Script Editor Institutional Version (RSEIV)

Following the retirement of RSE 1.0, Restudio Software Laboratories developed a new internal editor named Restudio Script Editor Institutional Version (RSEIV).

  • RSEIV is designed exclusively for industry-standard programming languages such as TSX, JavaScript, HTML, and related technologies.
  • RSEIV does not support Roland Script in any capacity.
  • RSEIV is an internal institutional product and is not publicly available. Access is granted only upon formal written approval from Restudio Support.

III. Practical Considerations

Zero User Adoption

Internal usage monitoring and project evaluation determined that Roland Script had no active user base and was not being used in production or personal software projects. Continuing investment in a project without practical adoption was therefore deemed technically and strategically unjustifiable.

Absence of an Ecosystem

Roland Script never developed a sustainable software ecosystem. No external libraries, community-driven packages, development tools, or active contributor community emerged around the language, preventing it from reaching long-term viability.

Impact of the Discontinuation

Effective immediately:

  • No further updates, bug fixes, security patches, or feature releases will be provided for Roland Script or Restudio Script Editor 1.0.
  • No Community Edition, Lite Edition, successor, or future continuation of Roland Script is planned.
  • Any continued use of the legacy interpreter or existing Roland Script code is undertaken entirely at the user's own responsibility, given the architectural limitations and technical deficiencies described in this statement.

Restudio Software Laboratories extends its appreciation to everyone who followed this experimental project during its development. The laboratories will continue focusing future efforts on modern software technologies built upon secure, maintainable, and industry-standard engineering principles.
